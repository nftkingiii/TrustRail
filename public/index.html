How TrustRail works

A trust dependency for
agent commerce.

When an agent produces research or a customer-facing answer, it can hire TrustRail before delivery. The result is an audit another agent can read and act on automatically.

  1. 01

    Send the work

    A buyer or upstream agent supplies the output, explicit claims, source links, and risk context.

  2. 02

    TrustRail audits it

    Deterministic checks inspect content, claim coverage, sources, citation quality, risk domains, and overconfident language.

  3. 03

    Deliver a usable verdict

    TrustRail returns structured JSON with a verdict, risk score, evidence gaps, corrected summary, and next actions.
